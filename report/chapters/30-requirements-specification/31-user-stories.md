# Capítulo III: Requirements Specification

## User Stories

En esta sección se definen los requisitos del ecosistema mediante una tabla única y consolidada de Épicas, User Stories de producto, User Stories del sitio web estático y Technical Stories del RESTful API. Los requisitos se estructuran a partir del Big Picture generado en nuestra sesión de Event Storming y del ajuste realizado en el solution profile y el needfinding, de modo que cada fila exprese un requisito real, comprobable y trazable. El lenguaje de las historias se mantiene en términos claros para el productor, el especialista y el visitante, evitando tecnicismos internos en las descripciones orientadas al usuario. En las historias dirigidas al usuario se describen acciones y resultados comprensibles; los detalles técnicos se reservan para las historias técnicas del backend, redactadas como necesidades del desarrollador sobre cada parte de la aplicación. Todos los criterios de aceptación se redactan en tiempo presente, tercera persona, bajo estructura BDD (Gherkin: Given-When-Then).

A continuación se presenta un único cuadro que dedica una fila a cada Epic / User Story, preservando la trazabilidad hacia el epic correspondiente y cubriendo tanto las necesidades del producto como las del sitio web informativo y el API.
